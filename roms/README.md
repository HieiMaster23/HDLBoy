# Test ROMs

This directory holds project-owned Game Boy ROM images used for FPGA bring-up.

## `minimal_led_blink.gb`

Generated by:

```text
python scripts/generate_minimal_led_rom.py
```

Purpose:

- 32 KiB no-MBC `ROM ONLY` cartridge image;
- jumps from both `0x0000` and the standard cartridge entry point `0x0100` to
  the same program at `0x0150`;
- repeatedly writes `0x05` and `0x0A` to `0xFF80`, allowing
  `sdram_cpu_rom_top` to prove that the CPU is executing code fetched from
  SDRAM by blinking the LED pattern.

Recommended loader smoke command:

```text
quartus_stp -t scripts/load_rom_virtual_jtag.tcl --dry-run --max-bytes 32 roms/minimal_led_blink.gb
```

## `minimal_visual.gb`

Generated by:

```text
python scripts/generate_minimal_visual_rom.py
```

Purpose:

- 32 KiB no-MBC `ROM ONLY` cartridge image;
- jumps from both `0x0000` and the standard cartridge entry point `0x0100` to
  the same program at `0x0150`;
- turns LCDC off while preparing video memory;
- writes tile 0, tile 1, and a 32x32 background map in VRAM;
- writes an alternating first tile row to the background map;
- configures `BGP = 0xFC`, `SCY = 1`, `SCX = 8`, and `LCDC = 0x91`;
- starts video through the normal LCDC enable path used by
  `sdram_video_rom_top`.

Recommended validation commands:

```text
cd sim/modelsim
vsim -c -do run_cpu_minimal_visual_rom.do
cd ../..
quartus_stp -t scripts/load_rom_virtual_jtag.tcl --dry-run roms/minimal_visual.gb
```

Expected hardware use:

```text
quartus_sh -t scripts/build_sdram_video_rom.tcl
quartus_pgm -m jtag -o "p;quartus/output_files/gameboy_core.sof"
quartus_stp -t scripts/load_rom_virtual_jtag.tcl --progress-step 4096 roms/minimal_visual.gb
```

If the SDRAM/video path is working, VGA should show the centered Game Boy
viewport with the first background row shifted by `SCX = 8` and using the same
white/checkerboard tile pattern as the earlier embedded visual demo.
