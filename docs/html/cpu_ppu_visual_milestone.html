Hardware visual milestone confirmed

CPU-to-PPU Visual Milestone

This report documents the first real board-level image generated from CPU-authored VRAM data and consumed by the first PPU background slice.

Validated top cpu_ppu_background_demo_top

Full chain present: CPU, bus, WRAM, VRAM, PPU, framebuffer, and VGA.

Observed image

Black border, bright centered Game Boy viewport, first tile row alternating white and checkerboard tiles.

Why it matters

The image is no longer direct framebuffer smoke output. It is produced by interacting hardware blocks.

Live Data Path

CPU
Writes tile data and tile-map bytes.
Bus
Decodes VRAM writes and debug marker.
VRAM
Stores 8 KiB of tile data and map entries.
PPU Slice
Reads tiles and expands them into pixels.
Framebuffer
Stores the rendered 160x144 image.
VGA
Scales and displays the final picture.

What This Proves

CPU-side proof The CPU can initialize video memory through the system bus.
Memory proof The VRAM block is exercised from both CPU and PPU sides.
PPU-side proof The renderer consumes tile data and tile-map contents correctly.
System proof The earlier VGA path still works after subsystem integration.
