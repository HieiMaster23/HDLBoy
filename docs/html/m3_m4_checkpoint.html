M3/M4 Checkpoint

This report documents the transition from the hardware-validated cpu_video_smoke_top to the first dedicated CPU memory-map module for the Game Boy FPGA core.

Validated top cpu_video_smoke_top

CPU writes visible pixels to the VGA framebuffer and displays 1234. Current SOF checksum: 0x00169AF4.

New module rtl/memory/bus_controller.vhd

Owns ROM decode, initial WRAM, I/O stubs, framebuffer write mapping, and pass/fail checks.

Regression run_cpu_video_smoke_top.do

Runs exact framebuffer-write checks plus the full top-level VGA smoke test.

Integration Flow

CPU
Sharp LR35902 subset fetches smoke ROM instructions.
Bus Controller
Maps ROM, framebuffer window, WRAM, HRAM, and I/O stubs.
Framebuffer
Stores 160x144 two-bit pixels in M9K RAM.
VGA Path
Reads pixels, scales 3x, drives 640x480 timing.
Debug Output
LED checkpoint and seven-segment 1234.

Memory Map

Range Current behavior Next step
0x0000..0x0118 Internal smoke-test ROM bytes. Replace with a cleaner ROM loader or generated ROM image flow.
0x8000..0xBFFF Experimental direct framebuffer write window. Split into VRAM and future PPU-owned framebuffer path.
0xC000..0xC03F Initial 64-byte WRAM page. Move to registered or block-RAM-backed storage before expanding.
0xE000..0xE03F Echo mirror for the initial WRAM page. Expand with WRAM when the bus supports lower-resource RAM.
0xFF00, 0xFF01..0xFF02, 0xFF04..0xFF07 Joypad, serial, and timer stubs. Replace with real input, serial, timer, and interrupt behavior.
0xFF40..0xFF4B LCD/PPU register stubs. Move ownership to the PPU implementation.
0xFF80 HRAM address overlaid by the debug LED checkpoint register. Remove debug overlay when a serial/test-output path exists.
0xFF81 HRAM address overlaid by the debug pass-code register. Reserve proper HRAM and add serial-style test output later.
0xFF0F, 0xFFFF IF and IE registers are implemented; lower five bits feed CPU interrupt ports. Connect interrupt dispatch once CPU servicing is implemented.

Process Steps

1

Close the checkpoint

Record the working CPU-to-framebuffer hardware test and keep the current top as the known-good integration target.

2

Organize debug artifacts

Move local SignalTap files to debug/signaltap/local/ and keep only documentation in source control.

3

Add regression

Validate the exact 64 framebuffer writes, final pass code, display value, and active VGA output in ModelSim.

4

Extract the bus

Move temporary ROM, I/O, and framebuffer decode into bus_controller.vhd.

5

Prepare M4

Add memory slices only when their Quartus cost is measured; full WRAM/HRAM requires registered reads or wait states.
